# ③ 토목기술팀

토목 설계·검토·계산 등 기술 업무 전담 부서.
CivilTools 앱의 검증된 모듈을 직원으로 하나씩 고정화한다(앱 껍데기는 버림).

## 폴더 구조

| 폴더 | 용도 |
|---|---|
| `00_result/` | 팀 **최종 결과물** 보관 (작업 1건 = `YYYY-MM-DD_HH-MM/`) |
| `01_data-download-desk/` | 직원 1 — 기상자료 크롬 다운로드 |
| `02_weather-desk/` | 직원 2 — 다운받은 파일 → 기상현황조사 Excel |
| `03_storm-rainfall-desk/` | 직원 3 — 다운받은 강우자료 → 분강우량·우량주상도 Excel |

## 업무 흐름

직원 1(다운로드)이 받은 자료를 직원 2·3이 작업 유형별로 이어받는다.

### 가. 기상현황조사 — 01 → 02

사용자가 **"토목기술팀, ○○ 관측소 △△년부터 10년간 기상현황 조사해줘"**:

1. **01_data-download-desk** — 기상자료개방포털(ASOS, pgmNo=36)에서
   월 자료 10년치 다운로드 (전제: 디버그 크롬 9222 + 포털 로그인)
2. **02_weather-desk** — 받은 파일을 파싱 → 기상현황조사 Excel 보고서 생성
3. 최종 결과물은 `00_result/`에 저장

→ 상세 절차는 `_작업흐름_기상현황조사.md` 참조.

### 나. 기왕호우·우량주상도 조사 — 01 → 03

사용자가 **"토목기술팀, ○○ 관측소 기왕호우·우량주상도 분석해줘 (호우일자들)"**:

1. **01_data-download-desk** — 포털(AWS, pgmNo=56)에서 사상별 분강우(분자료,
   하루 단위 3일)·시강우(시간자료) 다운로드
2. **03_storm-rainfall-desk** — 받은 파일을 사상별로 정리 → ① 분강우량 워크북
   ② 우량주상도 워크북 생성
3. 최종 결과물은 `00_result/`에 저장

→ 상세 절차는 `_작업흐름_기왕호우_우량주상도.md` 참조.

## 직원 예정
CivilTools의 다른 검증 모듈(확률강우량, 현황분석 등)도 같은 방식으로 직원화 예정.

## 주의
토목·설계 판단은 무조건 L3(사람이 결정). 직원은 자료수집·계산·보고서
초안까지만 하고, 기술적 적정성 판단은 사람이 한다.
