## 1단계 — 01_data-download-desk · 자료 다운로드

- **사이트**: https://data.kma.go.kr/data/grnd/selectAsosRltmList.do?pgmNo=36
- **🔑 작업 시작 전 — 로그인 먼저**: 다운로드를 시작하기 전에 사용자에게
  "그 크롬에서 data.kma.go.kr 로그인해 주세요"라고 요청하고, 사용자가
  "로그인했어"라고 확인할 때까지 대기한다. 확인 후에만 진행.
- **⚠️ 세션 만료**: 다운로드 중 "비회원 일부만 다운로드(31건)" 팝업이
  뜨면 = 세션 만료. 확인 누르지 말고 멈춰서 재로그인 요청.
- **전제**: 9222 디버그 크롬 실행 + data.kma.go.kr 로그인 상태
- **조회 조건 (고정값)**: 자료형태=월 자료 / 기간=최근 10년 /
  지점=요청 관측소 / 요소=전체

### 실행 순서 (검증됨 2026-05-18 — 이 순서대로 하면 빠름)
1. 페이지 이동 후 폼은 **네이티브 select**를 JS로 직접 설정한다
   (커스텀 UI라 click/fill이 안 먹음 → `evaluate_script` 사용):
   - `dataFormCd` = "월 자료" 옵션값 + change 이벤트
   - `startDt_y`/`startMt` = 시작연·월, `endDt_y`/`endMt` = 종료연·월
2. 지점: zTree에서 관측소 체크 (선택되면 hidden `stnIds`에 코드 들어감)
3. 요소: 요소 zTree(`ztree1`)에서 "전체" 노드 `checkNode(node,true,true)`
4. 조회: 폼 안 `onclick="goSearch()"` 버튼 → **`goSearch()` 호출**
   (호출하면 결과 페이지로 전환됨)
5. **⚠️ 용도신청 팝업** — 다운로드 클릭 시 `#wrap-datapop` 팝업이 뜬다.
   `해당 자료의 사용분야를 선택` → **라디오 `reqstPurposeCd` 중 "토목/건축"
   (value F00412) 체크** → "신청" 버튼 `fnRltmRequest()` 호출.
6. 다운로드: CSV는 `downloadRltmCSVData()`, Excel은 `downloadRltmXLSData()`.
   파일은 `C:\Users\user\Downloads`에 저장됨 (`OBS_ASOS_MNH_*.csv`).
7. **정리**: 받은 파일을 `01_data-download-desk/업무/YYYY-MM-DD_관측소_ASOS월자료_기간/`
   로 옮기고 이름 정리 + `다운로드기록.md` 작성.

> 참고: KMA ASOS 월자료 CSV는 **CP949 인코딩**, 컬럼 ~69개.

## 2단계 — 02_weather-desk · 결과물 생성

- **입력**: 1단계가 받은 ASOS 월자료 파일 (csv)
- CSV(CP949) 파싱 → 컬럼 헤더명으로 매핑 → `rows.json` 생성
  (`excel_export.py`가 먹는 형식: year/month/tempAvg/tempMax/tempMin/
  precipitation/windAvg/windMax/humidity/sunshine)
- CivilTools `excel_export.py` 실행 → 월별 표 + 연평균 + 차트 Excel
  ```
  py "E:/개인 FIle/01_Projects/02_hydro_work_auto/src/python/excel_export.py" rows.json 기상현황조사_관측소.xlsx
  ```
- **작업물(중간)**: `02_weather-desk/업무/YYYY-MM-DD_관측소_기간/`
  (parse.py · rows.json · 작업일지.md)

## 최종 결과물 — 원본 + 결과 함께 (중요)

`00_result/YYYY-MM-DD_HH-MM_요청명/` 폴더에 **반드시 셋 다** 넣는다:

| 파일 | 내용 |
|---|---|
| `기상현황조사_관측소.xlsx` | 결과물 (표·차트) |
| `관측소_ASOS_월자료_기간.csv` | **다운받은 원본 자료** |
| `작업일지.md` | 처리 내역·컬럼 매핑·요약 |

→ 사용자가 원본과 결과를 나란히 놓고 **2차 검증**할 수 있어야 한다.
   결과물만 주지 않는다. 원본 자료를 항상 동봉한다.

## 🔍 단계별 2차 검증 (필수 — 오류 없이 진행)

각 단계의 결과를 **다음 단계로 넘어가기 전에 반드시 확인**한다.
검증에 실패하면 멈추고 원인을 보고/재시도한다. 통과한 것만 다음으로.

| 단계 | 검증 항목 | 실패 시 |
|---|---|---|
| STEP1 지점 | 반환 `checked`에 요청 지점명(+코드)이 있는가 | 멈춤·재시도 |
| STEP2 조회 | `stnIds` 비어있지 않음 + `searched:true` | 멈춤·재시도 |
| STEP3 다운로드 | `biHweonWarning:false` | 세션만료 → 재로그인 요청 |
| 다운로드 파일 | CSV 행수 = 요청 개월수(10년→120) / 시작·끝 연월 일치 | 부분다운로드 의심 → 멈춤 |
| 02 파싱 | rows.json 개월수 = CSV 데이터행수 / 컬럼매핑 N/A 없음 | 매핑 재확인 |
| 02 Excel | `excel_export.py` 반환 `success:true` | 재실행 |
| 최종 | `00_result` 폴더에 원본CSV + 결과xlsx + 작업일지 3개 다 존재 | 누락분 보완 |

추가로 결과 수치가 상식적인지 본다 — 연평균기온이 한반도 범위(약 5~16℃),
연강수량(약 900~1700mm) 안인지. 벗어나면 컬럼매핑·파싱 오류 의심.

## 직원 분담 원칙

- 최종물은 00_result로, 원본 자료를 동봉.
- 새 작업 유형이 생기면 이 폴더에 `_작업흐름_*.md`를 추가한다.
