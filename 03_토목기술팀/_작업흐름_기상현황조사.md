# 작업 흐름 — 기상현황조사 (토목기술팀)

> 이 파일은 "기상현황조사" 작업의 **표준 흐름**이다.
> 사용자가 아래 트리거 문장을 말하면 00 → 01 직원이 이 순서대로
> 분담 실행한다. (메인 Claude가 두 직원 지침을 따라 직접 실행)

## 트리거

> "토목기술팀, **○○ 관측소 최근 10년 기상현황 분석해줘**"
> (예: "문경 관측소 2025년 기준 최근 10년 기상현황 분석해줘")

입력값은 **관측소명**과 **기간(최근 N년)** 둘뿐. 나머지는 아래 고정값.

---

## 1단계 — 00_data-download-desk · 자료 다운로드

- **사이트**: https://data.kma.go.kr/data/grnd/selectAsosRltmList.do?pgmNo=36
  (종관기상관측 ASOS — 자료)
- **전제**: 9222 디버그 크롬 실행 + data.kma.go.kr 로그인 상태
- **조회 조건 (기상현황조사 고정값)**:
  | 항목 | 값 |
  |------|-----|
  | 자료형태 | **월 자료** |
  | 기간 | 최근 10년 (예: 2016.01 ~ 2025.12) |
  | 지점 | 요청한 관측소 (예: 경상북도 문경) |
  | 요소 | 기온 · 강수량 · 풍속 · 습도 (또는 전체) |
- 조회 → 다운로드(CSV 또는 Excel 버튼) → 받은 파일을 업무폴더로 정리
- **저장**: `00_data-download-desk/업무/YYYY-MM-DD_관측소_ASOS월자료_기간/`
  - 받은 원본 파일 + `다운로드기록.md`

## 2단계 — 01_weather-desk · 결과물 생성

- **입력**: 1단계가 받은 ASOS 월자료 파일 (csv 또는 xlsx)
- 파일 파싱 → rows JSON → CivilTools `excel_export.py` 실행
- → 월별 표 + 연평균 + 차트가 든 기상현황조사 Excel 보고서
- **저장**: `01_weather-desk/업무/YYYY-MM-DD_관측소_기간/`
  - `기상현황조사_관측소.xlsx` + `rows.json` + `작업일지.md`

## 결과물

`01_weather-desk` 업무폴더의 **기상현황조사 Excel 보고서**.

## 직원 분담 원칙

- 00은 **받기만**, 01은 **만들기만**. 서로 영역 침범 안 함.
- 00이 받은 파일을 01이 입력으로 받는 릴레이.
- 새 작업 유형이 생기면 이 폴더에 `_작업흐름_*.md`를 추가한다.
