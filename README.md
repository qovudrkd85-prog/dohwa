# 00_dohwa — 도화 검토 오피스

회사 업무를 부서별 에이전트로 처리하는 "에이전트 오피스"의 작업 폴더.

## 폴더 구조 (부서 → 직원 → 업무)

```
00_dohwa/
├── 도화회사 조직도.bat     ← 더블클릭하면 조직도 화면이 뜸
├── 조직도.html             ← 조직도 화면 (보기 전용)
├── 그루웨어-크롬-실행.bat   ← 그루웨어 자동화용 크롬 실행기
│
├── 01_행정팀/
│   └── schedule-desk/        직원
│       └── 업무/브리핑/      ← 작업물
├── 02_문서팀/                (직원 예정)
├── 03_토목기술팀/                기상현황조사 파이프라인
│   ├── 00_result/              팀 최종 결과물 폴더
│   ├── 01_data-download-desk/  직원 1단계 (포털 크롬 다운로드)
│   └── 02_weather-desk/        직원 2단계 (파일→Excel 분석)
├── 04_법무인허가팀/          (직원 예정)
├── 05_데이터팀/              (직원 예정)
├── 06_검토팀/
│   ├── simui-desk/           직원
│   ├── hwp-review-desk/      직원
│   └── ppt-review-desk/      직원
│       └── 업무/             ← 작업물
│
└── QBS/                      독립 프로젝트 폴더 (오피스 구조와 별개)
```

- **부서 폴더**: `NN_부서명/` — 각 폴더에 `_부서소개.md`
- **직원 폴더**: `부서/직원id/` — 각 폴더에 `_직원소개.md`
- **업무 폴더**: `직원/업무/` — 작업물이 쌓이는 곳.
  검토팀은 작업 1건 = `업무/YYYY-MM-DD_사업명/` 폴더 하나.

## 부서 현황 (Claude Code 에이전트, `~/.claude/agents/`)

| # | 부서 | 직원 | 상태 |
|---|------|------|------|
| ① | 행정팀 | schedule-desk | 가동 |
| ② | 문서팀 | - | 직원 예정 |
| ③ | 토목기술팀 | data-download-desk · weather-desk | 가동 |
| ④ | 법무인허가팀 | - | 직원 예정 |
| ⑤ | 데이터팀 | - | 직원 예정 |
| ⑥ | 검토팀 | simui-desk · hwp-review-desk · ppt-review-desk | 가동 |

## 쓰는 법

- **회사 구조 보기**: `도화회사 조직도.bat` 더블클릭
- **일 시키기**: VSCode(Claude Code)에서 직원 호출 문구 입력
  - "오늘 정리해줘" / "한글 보고서 검토해줘" / "PPT 검토해줘" / "심의의견 정리해줘"
- 작업 결과는 해당 직원의 `업무/` 폴더에 쌓인다.

## 운영 규칙

- 조직도(.bat/.html)는 **보기 전용** — 거기서 일은 못 시킨다. 일은 VSCode에서.
- 그루웨어는 동시접속 1개 → .bat 크롬 쓸 땐 다른 곳 로그아웃
- 메일 발송·일정 확정·기술 판단 = 사람이 결정 (에이전트는 "검토 바랍니다"로 멈춤)
- 직원이 늘거나 바뀌면 Claude에게 "조직도 갱신해줘"

> 프로젝트 상세 이력은 Claude 메모리 `project_agent_office` 참고.
