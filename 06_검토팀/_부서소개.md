# ⑥ 검토팀

보고서·발표자료·심의자료의 변경·검토의견을 시각화하는 전담 부서.

## 소속 직원 (3명)
| 직원 폴더 | 역할 | 호출 문구 |
|---|---|---|
| `simui-desk/` | 심의검토원 — 위원 HWP 조치계획서 → 보고용 PPT 표 반영 | "심의의견 정리해줘" |
| `hwp-review-desk/` | 한글검토원 — 두 한글 문서 비교 → 색칠 검토본 | "한글 보고서 검토해줘" |
| `ppt-review-desk/` | PPT검토원 — 두 PPT 비교 → 주석박스 검토본 + 변경요약 | "PPT 검토해줘" |

## 폴더 규칙
- 직원별 폴더 안 `업무/` 에 작업물이 쌓인다.
- 작업 1건 = `업무/YYYY-MM-DD_사업명_작업종류/` 폴더 하나.
  - 그 안에 입력 원본 / 중간 결과 / 산출물(검토본) / 작업일지.md 를 모은다.

## 업무 수준
세 직원 모두 L2 — 검토본 초안까지 만들고, 변경·의견의 기술·토목 타당성
판단(L3)은 사람이 한다.
